# Deployment

## Generate Artifacts

```bash
python3 scripts/scrape_sources.py
```

The canonical Snowflake load artifact is:

```text
artifacts/processed/documents.jsonl
```

## Snowflake

Upload `artifacts/processed/documents.jsonl` to:

```text
@BREAST_CANCER_SEARCH.RAG.BREAST_CANCER_STAGE
```

Run the Snowflake setup in order:

```sql
sql/01_create_database_schema.sql
```

Upload `artifacts/processed/documents.jsonl` to the stage, then continue:

```sql
sql/02_create_raw_documents_table.sql
sql/03_load_raw_documents.sql
sql/04_create_document_chunks.sql
sql/05_create_cortex_search_service.sql
sql/06_validate_app_dependencies.sql
```

Expected checkpoint after `03_load_raw_documents.sql`: `RAW_DOCUMENTS` should contain the number of rows in `artifacts/processed/documents.jsonl`.

Expected checkpoint after `04_create_document_chunks.sql`: `DOCUMENT_CHUNKS` should contain non-empty search chunks generated by `SNOWFLAKE.CORTEX.SPLIT_TEXT_RECURSIVE_CHARACTER`.

## Streamlit In Snowflake

Use `app/streamlit_app.py` as the canonical app file. The root `streamlit_app.py` wrapper is retained only as a convenience entrypoint.

`app/streamlit_app.py` imports reusable search and generation helpers from
`src/snowflake/cortex.py`. When deploying to Streamlit in Snowflake, include
the `src/` package alongside the app file instead of uploading only
`app/streamlit_app.py`.

Minimum application package layout:

```text
app/streamlit_app.py
src/
```

Required packages:

- `streamlit`
- `snowflake-snowpark-python`
- `snowflake`

The active role must have access to the database/schema, Cortex Search service, and the `SNOWFLAKE.CORTEX_USER` database role.
